## üõ†Ô∏è How to Contribute

### Reporting Bugs
If you find a bug, please create a GitHub Issue with:
*   A clear title description.
*   Steps to reproduce.
*   The expected vs. actual behavior.
*   Screenshots if applicable.

### Suggesting Features
We love new ideas! Please open a GitHub Issue with the "enhancement" label and describe your idea in detail.

### Pull Requests
1.  **Fork** the repository.
2.  Create a **new branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
3.  Make your changes.
4.  Run the app locally to ensure everything works (`npm start`).
5.  **Commit** your changes with a clear message.
6.  Push to your fork and submit a **Pull Request**.
